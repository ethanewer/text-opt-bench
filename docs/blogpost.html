How well does an LLM optimize text?

text-opt-bm hands an LLM agent a weak starting program and a scoring function, then measures how far it can drive the score down by iteratively rewriting the program — across memory, compression, algorithms, and learning-from-data tasks.

13 tasks4 families no wall-clock in scoringbit-for-bit replayable gpt-5.5 · 1-hour budget

The benchmark

An agent (a codex loop) repeatedly rewrites a program to minimize a score. Scores are byte counts, memory (tracemalloc), executed-bytecode counts, or error rates — never wall-clock — so results are hardware-independent and replay exactly. Eight perfect-information tasks are scored directly on the workload they optimize. Five generalization tasks are graded on a fully-visible training set but carry a large hidden test set, drawn from the same distribution and never shown, that measures whether the program generalizes rather than memorizes.

  • memory compact data structures scored by serving-time memory — mem_kv / index / intset / str / infer
  • compression a lossless codec scored by output size — compress
  • algorithms scored by executed bytecode / recompute cost — ops_connect, checkpoint_plan
  • generalization learn from visible train, scored on a hidden test — word_problems, normalize, rule_list, tag_seq, compress_heldout

Experiment 1 · b

On the generalization tasks, the hidden test tells the real story

The five generalization tasks are graded on the visible training set, which the agent drives to essentially zero error at every effort. What differs is the hidden held-out test (below): on word_problems and normalize the program generalizes; on rule_list and tag_seq it overfits the visible data (a deep model fits the train labels, including noise, and the test error stays high).

Experiment 2

Hiding the evaluation data can help

Does letting the agent see the data it is graded on help or hurt? Each generalization task is run two ways, both measured on the same hidden test: visible — the graded set is the fully-visible training data (the agent can fit or memorize it); and hidden — the agent sees only 5 example rows and an aggregate validation score on a set it never sees, so it cannot memorize. On the tasks prone to overfitting, hiding the data yields equal or better test error.

Experiment 3

How much does training-set size matter?

Holding everything else fixed, the visible training set shrinks from 1:4 of the test down to 1:8 and 1:16. More training data helps most where the program actually learns from examples (word_problems); where it overfits or relies on format breadth, size matters little.
