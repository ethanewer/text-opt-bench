text-opt-bm

A deterministic benchmark measuring how well an LLM agent optimizes text — code, prompts, codecs, data structures — by iteratively rewriting a program to minimize a score.

The benchmark

Each task gives an LLM agent (a codex loop) a weak starting program and a scoring function; it edits the program over many iterations to drive the score down. Scores are byte counts, memory (tracemalloc), executed-bytecode counts, or error rates — never wall-clock — so runs are hardware-independent and replay bit-for-bit. 14 tasks in four families:

  • memory compact structures scored by serving-time memory — mem_kv/index/graph/intset/str/infer
  • compression lossless codecs scored by output size — compress, compress_heldout
  • algorithms scored by executed bytecode — ops_connect, checkpoint_plan
  • generalization learn from a visible train split, scored on hidden val/test sealed with a secret seed — word_problems, normalize, rule_list, tag_seq, compress_heldout

Experiment 2 — optimization overfits unless the eval data is hidden

The 5 generalization tasks, run two ways scored on the same val instances — hidden-val (agent sees only the val score) vs exposed-val (val data moved into the visible training set) — each also judged on an always-hidden test split. For every plot: train (val) score and held-out test score over time.

A visible eval score is untrustworthy — the optimizer fits it perfectly without generalizing. Largest where data is idiosyncratic/memorizable (rule_list, compress_heldout: test rises), smallest on the solvable word_problems (more data helps). The control is structural: hide the eval data and expose only an aggregate score — every generalization task seals val/test with a secret seed.
