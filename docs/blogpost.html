text-opt-bm

Benchmarking LLMs on text optimization

text-opt-bm gives an LLM agent a weak Python program and a scoring function. The agent repeatedly edits the program, and the benchmark records the best valid score it finds.

  • scores Metrics include traced memory, compressed bytes, executed bytecode, recompute cost, and error rate.
  • perfect information Eight tasks are scored on the same workload the agent optimizes.
  • generalization Five tasks expose a training set and keep a larger sealed test from the same distribution.
  • harder tasks Three ML-systems tasks add routing, optimizer transfer, and physically constrained model compression with validation-guided selection and sealed tests.
  • plots Curves step at accepted submissions. Select any task card for its objective, metric, and exact data split.

Experiment 4 · harder tasks

Research-oriented ML systems optimization

The original suite emphasizes compact algorithmic programs. The three tasks below move closer to research workflows: they use larger, explicitly separated development and test sets; score validation-selected incumbents on sealed data; and expose metrics that correspond to real deployment or training costs. All counts below come from the active benchmark manifests.

  • selection Visible training or calibration data may be used to build a method, while acceptance is validation-guided. Sealed tests cannot affect acceptance or later agent prompts.
  • budget Runs receive one hour of active optimization time. Evaluation-capacity intervals are excluded from charged time, and final sealed-test work drains after optimization.
  • reporting Lower is better for all three primary metrics. Test results belong to validation-selected programs, not test-selected checkpoints.
  • agent gpt-5.6-sol with high reasoning; five independent trials per task.
  • time axis Active optimization time from 0 to 60 minutes; evaluator queueing is refunded.

Curve semantics: as in the earlier figures, faint horizontal/vertical step traces are independent trials and the bold horizontal/vertical trace is their pointwise N=5 mean. Every point represents a validation-selected incumbent. Its train score may rise because train performance does not control acceptance. A holdout trace changes only when that exact submitted program has a sealed score; sparse deferred tests therefore remain flat between scored incumbents. Horizontal references are locally evaluated baselines. Time is active optimization time, with evaluation-capacity intervals excluded.

Experiment scope. Experiments 1a–3 use gpt-5.5 at high, low, and no reasoning effort plus grok-4.5 xhigh where shown. Experiment 4 uses gpt-5.6-sol with high reasoning on routing-v6, optimizer-v8, and LFM-compression-v2. Every plotted condition has five independent trials and a one-hour optimization budget. Aggregate bands show one standard deviation over independent run choices. Separate saturation probes are excluded. text-opt-bm is a work in progress.
