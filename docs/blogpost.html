How well does an LLM optimize text?

text-opt-bm hands an LLM agent a weak starting program and a scoring function, then measures how far it can drive the score down by iteratively rewriting the program — across memory, compression, algorithms, and learning-from-data tasks.

13 tasks4 families no wall-clock in scoringbit-for-bit replayable gpt-5.5 · 1-hour budget

The benchmark

An agent (a codex loop) repeatedly rewrites a program to minimize a score. Scores are byte counts, memory (tracemalloc), executed-bytecode counts, or error rates — never wall-clock — so results are hardware-independent and replay exactly. Eight perfect-information tasks are scored directly on the workload they optimize. Five generalization tasks are graded on a fully-visible training set but carry a large hidden test set, drawn from the same distribution and never shown, that measures whether the program generalizes rather than memorizes. Every curve below is a step function: each corner is one accepted submission.

  • memory compact data structures scored by serving-time memory — mem_kv / index / intset / str / infer
  • compression a lossless codec scored by output size — compress
  • algorithms scored by executed bytecode / recompute cost — ops_connect, checkpoint_plan
  • generalization learn from visible train, scored on a hidden test — word_problems, normalize, rule_list, tag_seq, compress_heldout

The same model, gpt-5.5, runs at three reasoning efforts — high, low, and none — 5 runs each, 1-hour budget. Experiment 1b

Experiment 2

Hiding the evaluation data can help

Does letting the agent see the data it is graded on help or hurt? Each task is run two ways, both measured on the same hidden test: visible — the graded set is the fully-visible train (the agent can fit/memorize it); and hidden — the agent sees only 5 example rows and an aggregate validation score on data it never sees. Below: train, the hidden validation score, and the held-out test. On overfitting-prone tasks, hiding the data gives equal or better test error.

Experiment 3

How much does training-set size matter?

Holding everything else fixed, the visible training set shrinks from 1:4 of the test down to 1:8 and 1:16. More data helps most where the program actually learns from examples (word_problems); where it overfits or relies on format breadth, size matters little.
